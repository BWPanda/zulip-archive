ericfg (Mar 06 2020 at 17:47):

from a complex multilingual site we maintain:

  • Administration language (admin_language)
  • Block languages (i18n_block)
  • Contact translation (i18n_contact)
  • Field translation (i18n_field)
  • Menu translation (i18n_menu)
  • Multilingual content (i18n_node)
  • Multilingual forum (i18n_forum)
  • Multilingual select (i18n_select)
  • Path translation (i18n_path)
  • Rules translation (rules_i18n)
  • String translation (i18n_string)
  • Synchronize translations (i18n_sync)
  • Taxonomy translation (i18n_taxonomy)
  • Translation redirect (i18n_redirect)
  • Translation sets (i18n_translation)
  • User mail translation (i18n_user)
  • Variable translation (i18n_variable)
  • Views translation (i18nviews)

I have not tried to move this site to backdrop yet so I can't speak to the needs, but want to note that the client that this site is for has been unhappy with i18n in general in Drupal7; they have been told that D8 sorts this out better and I am hoping that by the time they are ready to do something new, Backdrop will also do this better.
So, my question; has much/anything changed between D7 and Backdrop in how i18n works?
