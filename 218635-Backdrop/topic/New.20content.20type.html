Personally, I would love to see a generic "type" that can have an alias path field added to it. When I brought this up, I was told there is too much old code that would cause major rewrites to have to happen so it was pretty much a hard no.

But anywho, I do like the approach Drupal took with converting blocks to entities which allows fieldable blocks. I think Backdrop makes it a LOT easier to handle fieldable content with layouts though so a content type could always be used for this proposed purpose. I guess I'm good either way?
