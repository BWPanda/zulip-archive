bugfolder (Nov 22 2020 at 19:05):

I am working on porting the Webform2PDF module (port in progress is here). This module uses the open-source TCPDF library. in the Drupal version of the module, TCPDF lived in the /libraries folder; in Backdrop, though, we are supposed to bundle libraries within the module.

Now, Webform2PDF is GPLv2, while TCPDF is LGPLv3. I'd like to double-check my assumptions with anyone here who knows the proper way of dealing with this issue about the proper thing to do in the distribution.

My assumption is that it's OK to organize the module like this:

webform2pdf
    libraries
        tcpdf
            ...files...
            LICENSE.txt (LGPLv3)
    webform2pdf.module
    ...more module files...
    LICENSE.txt (GPLv2)
    README.md

And then in the README.md (which you can see here) list the two licenses separately.

Is that the right way to deal with this situation?

klonos (Nov 22 2020 at 21:10):

...this is already documented: https://github.com/backdrop-ops/contrib#new-project-checklist

  • If your project includes a library that is licensed under GPL-3.0, LGPL-3.0, Apache-2.0, AGPL-3, or any other license that the Free Software Foundation has determined to only be compatible with GPL-3.0, then you must license your project as GPL-3.0. Please use this text for GPL-3.0.
  • The GPL-3.0 license will prevent your project from one day being included in Backdrop core, so please only use this license when necessary.
backdrop-ops/contrib
Apply to join the contributed code developer team. - backdrop-ops/contrib

klonos (Nov 22 2020 at 21:13):

http://www.gnu.org/licenses/license-list.html#GPLCompatibleLicenses lists LGPL v3 as GPL-compatible, so there you have it: TCPDF can be bundled with the module, and the entire module may be licensed as GPL v3, and https://raw.githubusercontent.com/backdrop-ops/contrib/master/examples/LICENSE-GPL-3-0.txt should be used instead.
