indigoxela (Jan 27 2021 at 09:11):

I wonder, how we can deal with potential security issues in contrib modules.

A real life example: https://github.com/backdrop-contrib/ckeditor_addon

This contrib module ships with an outdated ckeditor version (ships with 4.5.7 but 4.15.1 was a security release). Note that there has never been a stable release and the module seems abandoned, but people might have installed (or might still be installing) the dev version. There is no warning, when doing so.

What's the procedure to handle that? Is there any?

backdrop-contrib/ckeditor_addon

BWPanda (Jan 27 2021 at 09:13):

@indigoxela https://backdropcms.org/security

BWPanda (Jan 27 2021 at 09:13):

If you have found a security issue in Backdrop Core or a Contributed Project, send an email to security@backdropcms.org. Do not file an issue on GitHub if you think you have encountered a security issue. The Backdrop Security Team uses a private repository on GitHub where security issues are discussed and managed. When a security issue that affects Backdrop CMS is resolved, a release will come out on the following Wednesday.
