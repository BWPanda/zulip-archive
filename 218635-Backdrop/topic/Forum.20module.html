jenlampton (Mar 10 2021 at 23:18):

Hm, does it make even more sense to build a forum with site building tools? Basically a vocabulary for different Forums, a custom content type for Topics, comments for Replies, and Views to display various lists?

@Olaf Grabienski This is exactly what we did with the forum site. The forum module was in core before core had views. With taxonomy, node types and views, my argument was that we didn't need a module to make a forum anymore, and instead, we should make sure that core could do everything in forum module.

jenlampton (Mar 10 2021 at 23:23):

Maybe we should release our version as an alternative solution...

@klonos @BWPanda The custom borg_forum module on b.org is not a replacement for the old forum module (since, we don't need a forum module anymore) instead, it's "a collection of all the misc stuff on the forum site". Most of my sites have one of these. If the site were named GobbledyGook, the module would have been mamed "borg_gobbledygook". Sorry for the confusion!
