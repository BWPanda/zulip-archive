esn (Nov 26 2020 at 06:46):

@oadaeh (Jason Flatt)
Thank you for your suggestion.
I was able to resolve the issue by reading that webserver log.
The solution is as follows ...

Error log stated
"SoftException in Application.cpp:267: File "/home2/xxxx/public_html/xxxx/index.php" is writeable by group"

But when I looked at the permissions of index.php the group write option was not selected.
The permission level was 644

So I selected all the permission options (The permission level was now 777) then saved the file.
Re-ran the install again (Was just checking), it still had the same error, as expected.
I then set the permission back to 644 but this time when I ran the install, it worked perfectly.

So clearly there was a problem with the webserver, for some reason
it kept reading install.php was writable by group, even though it wasn't.
So changing the permission to something else seemed to unlock its stuck state.
I probably would have only needed to set the group write option save it then un-set it to release the web-server's stuck read state.

Thank you for all contributing, it's greatly appreciated.
