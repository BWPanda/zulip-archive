I'm going to paste my notes from the session. Here's my paraphrasing of the comments people posted:

article:
https://www.freeform.ca/upgrading-website-drupal-7-backdrop-cms

--
Fixing links or file paths that are embedded in body fields:
• https://github.com/interconnectit/Search-Replace-DB
• port https://www.drupal.org/project/scanner
• Pathologic (already exists in Backdrop)
• make a symlink

cd sites/default
ln -s ../../files files

--
If you are using feeds to migrate - you can use feeds tamper to do some of this as well.

In more complex sites, we migrate using feeds to bring in content rather than upgrade. I have found it to be a better solution.

--
Watch out for dates!
And files and images when migrating ...
file and image fields, not in body

--
Here’s a Drupal 7 module that will tell you the status of all your Drupal 7 modules for Backdrop: https://github.com/backdrop-contrib/backdrop_upgrade_status

interconnectit/Search-Replace-DB
This script was made to aid the process of migrating PHP and MySQL based websites. Works with most common CMSes. - interconnectit/Search-Replace-DB
backdrop-contrib/backdrop_upgrade_status
Checks to see if the installed modules on a Drupal 7 site are ready for upgrade to Backdrop CMS. - backdrop-contrib/backdrop_upgrade_status
