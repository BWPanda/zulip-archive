klonos (May 17 2020 at 16:00):

I've seen this:

  $config = config('some_config.settings');
  $config->set('some_setting', $form_state['values']['some_setting']);
  $config->set('another_setting', $form_state['values']['another_setting']);
  $config->set('yet_another_setting', $form_state['values']['yet_another_setting']);
  $config->save();

...but also this:

  $config = config('some_config.settings');
  $config
    ->set('some_setting', $form_state['values']['some_setting'])
    ->set('another_setting', $form_state['values']['another_setting'])
    ->set('yet_another_setting', $form_state['values']['yet_another_setting'])
    ->save();

Are these both acceptable?

oadaeh (Jason Flatt) (May 17 2020 at 16:35):

Yes, if the return value of the method (set()) is an object. The following method needs to be valid for the returned object.

oadaeh (Jason Flatt) (May 17 2020 at 16:39):

Config::set() returns $this, which is a pointer to itself (a config object), so you can chain calls to the method: https://api.backdropcms.org/api/backdrop/core%21includes%21config.inc/function/Config%3A%3Aset/1
