BWPanda (Sep 29 2020 at 00:17):

Well destroying won't do anything if you've only just set it up for the first time... Here's a list of things to make sure you do:

  • Clone repo (your fork or the main one, just make sure it's up-to-date)
  • Setup your .env file (for accessing sanitize.backdropcms.org)
  • Start Lando
  • Run lando pulldb to get the latest DB
  • Run lando php scripts/lp-pull.php --files to get the latest files
  • Manually unzip and move them to the first place (I aim to automate this a bit more, see https://github.com/backdrop-ops/api.backdropcms.org/issues/91)
  • You might need to add the Lando URL to settings.php for the 'trusted_host_patterns' thing
  • Copy all live-active config to staging and dev-active (normally you'd run drush bcim to update dev-active, but I couldn't run that because it was empty and wouldn't work until I did it manually first)
  • Access site and make sure everything works (on the main branch)
  • Switch to issue branch
  • Run lando drush bcim to import issue-specific config
  • Run lando drush cc all just for the heck of it
  • Make sure issue-branch works as expected
Add Lando tooling to allow pulling the latest DB/files 路 Issue #91 路 backdrop-ops/api.backdropcms.org
I'd like to do the same here as we did in backdrop-ops/backdropcms.org#678 - adding a Lando command for pulling in and setting up the latest API website DB/files.
