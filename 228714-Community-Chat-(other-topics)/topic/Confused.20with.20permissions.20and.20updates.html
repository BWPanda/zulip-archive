Phil Ward (Mar 26 2021 at 18:53):

1) looser permissions are only dangerous if you get hacked when using a dedicated hosting environment. Personally, I don't recommend loosening permissions, but rather figure out why the correct permissions aren't working.

2) the user:group on your server is usually setup that way on account of how your systems webserver wants to handle permissions. The www-data group is usually for the webserver so it has permission to read/write to the hosting directory. Other server configurations will lock down the group to the user, then add the webserver into each users group. (I believe that's how it works... I'm not a Linux sysadmin)

3) I can't say why the update isn't working and calling for FTP. I ran into this years ago on a Drupal install using simplytest.me and a 3rd party hosting environment for a client. After that, I quit using the GUI for updates because if there are too many inconsistencies where some hosting environments will allow it and others won't, it's not worth my time to figure it out, so I went the manual route until I discovered drush for command line.

Keep in mind, Backdrop is still Drupal under the hood. When running into problems like these, you can always do a search and replace Backdrop with Drupal.

Might try this: https://www.drupal.org/forum/support/post-installation/2011-01-22/module-installation-requires-ftp-access#comment-4690116
