Laryn said:

I have a long-term goal of getting Breakpoints and Picture ported, but in the meantime this is a DIY implementation of <picture> functionality, so on small screens it uses one image style and on larger screens it uses another.

If you do manage to tackle this, just a thought... Maybe re-work how breakpoints work from a UX standpoint? I love its purpose, but never use it because of its design. I always felt it was too cumbersome to setup and maintain. The UX just didn't jive with me. I felt like it should be simple to use, but couldn't fully wrap my head around how to use it. With the advancements of Flex and Grid, I kind of feel like breakpoints should be relegated specifically to images as opposed to themes or layouts though I do know there might still be a place for those. All I ever really wanted from breakpoints was to use it for image file size control based on viewport. (specifically smaller file sizes on mobile)

What I would suggest, is move the breakpoints to the image styles themselves instead of using it global for the site.

Totally hijacked this topic, so we can start a new one if you're interested in collaborating more.
