klonos (Mar 19 2021 at 21:24):

I wasn't able to find any existing issue for this, but would be very odd if nobody noticed it: when you create/edit taxonomy terms, you can add a description, and that description uses CKEditor, with the ability to add images. Bugs:

  • you may select an existing image, but there is no option in the "Insert image" dialog to upload a new image :thumbs_down:
  • existing images can be added to the description of terms, and they can be viewed, so they are saved properly, but the usage count for the image file does not get incremented. This means that if the "main" image gets deleted from everywhere else it is being used, eventually its usage count will reach 0, despite the fact that it is being used in one or more term descriptions ...eventually, garbage collection will kick in, and it will delete the actual file, which will result in data loss :thumbs_down: :thumbs_down:

So, is there any existing issue for this that I couldn't find?
