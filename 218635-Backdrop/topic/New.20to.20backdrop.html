klonos (Jul 15 2020 at 14:45):

Hey @sdubois :wave: ...the TL;DR answer is that no, porting _simple_ D7 modules in Backdrop is not a huge deal at all. You can actually just change a line or 2 in the .info file of the D7 module, and chances are that it might just work like that. This does not apply to complex modules like Rules, or Webform for example. ...there is some work required, but that is usually converting things to use configuration management. ...there's also https://github.com/backdrop-contrib/coder_upgrade which you can use to run the D7 module through, and coder_upgrade will automatically attempt to fix/port certain things for you.

BWPanda (Jul 15 2020 at 21:17):

@sdubois If you decide to port a D7 module to Backdrop, there's no requirement for you to continue maintaining it in the future. Obviously that's preferred, but you can just add a note to the README that the project is looking for maintainers and that's fine.

jenlampton (Jul 15 2020 at 21:17):

@sdubois to answer your question about contrib: no - you are not entirely responsible for maintaining a module after you have ported it from Drupal. You need to be willing to work with the security team if there is an issue, but that does not mean that you need to do all the changes yourself, there are people here to help you. We have both a "Bug Squad" that can help fix problems, and a "Security team" that have commit access to all Backdrop contrib repositories.

jenlampton (Jul 15 2020 at 21:17):

jynx ;)

jenlampton (Jul 15 2020 at 21:21):

@sdubois and regarding Twig, I don't think anyone is using it with Backdrop, but there's no reason the D7 module wouldn't work in exactly the same way. IIRC that module will only use Twig when its own templates replace core's, so all templates (including layouts) would still use the PHPTemplate version until overridden.

jenlampton (Jul 15 2020 at 21:23):

That said, we did apply some of the same theme system simplifications to Backdrop that went into Drupal 8 -- but we did what we could while maintaining as much backwards compatibility as possible (so, without Twig). I'm not sure why Twig was chosen for you project, but you may find that the changes included in Backdrop are enough of an improvement without it.
