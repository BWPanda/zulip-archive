oadaeh (Jason Flatt) (Jan 07 2020 at 13:32):

That separate contrib documentation section has existed for a long time. As a Drupal project maintainer, keeping up on documentation has always been a sore spot for me. As a Drupal project maintainer, I am supposed to keep up to date separate documentation for each project in three areas: the README.txt file, the main project page, and that contrib documentation section. And even if they all had the same content, you still have to keep up on the three separate areas, and the formatting for at least two of them is different. For someone who does not have a lot of time for responding to and fixing issues, maintaining three separate sets of documentation is a little too much.

oadaeh (Jason Flatt) (Jan 07 2020 at 13:41):

Something nice about using the README for the project page is that key information can be included and people can either view it locally or navigate to the project page, whichever is more convenient for the user at the time. I always found it annoying (but totally understandable, given the documentation situation) to be reading the README for information on installation and/or configuration only to be pointed to a website. It kind of made the README seem pointless.
