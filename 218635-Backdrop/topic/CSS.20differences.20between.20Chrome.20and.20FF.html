BWPanda (May 11 2020 at 13:59):

Ok, after the above 'fix', I experienced further, similar problems with different parts of the site, but the above 'fix' didn't work for those. I'm thankful to sya however that I found and fixed the root cause and that my site now looks proper in Firefox and Chrome. The issue was that I'm using Seven as my base theme (it has nice form styling, etc.) and it has CSS which displays blocks as tables:

.block {
  display: table;
}

(I don't know why...)
So wherever I had blocks as flexbox items, it was mucking up in FF. I simply changed blocks in flexbox to display as blocks and that fixed it:

.wrapper {
  display: flex;

  .block {
    display: block;
  }
}

(I use SASS - gotta love that indenting!)
