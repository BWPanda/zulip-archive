indigoxela (Mar 16 2021 at 06:40):

I prefer to drop a message here, as I'm not planning to further contribute to the issue

Important things from my perspective:

  • Do not complicate D7 migrations - people might have used contrib modules for meta tags and the migration path could get cumbersome if parts are in core and other parts aren't
  • Do not force module maintainers to fundamentally rewrite their code within 1.x (remember the Backdrop philosophy regarding contrib)
  • Do not assume that all sites need extra metatags (all those og:* and twitter:* tags might - worst case - be undesirable)
  • Do not assume what sorts of pages, layouts, views, content types, ... sites have - this vastly varies because of the flexibility of Backdrop
  • Do not add meta tags OOTB, that need extra configuration - bad/wrong meta tags can cause a regression (all of the proposed extra meta tags need proper configuration, btw)

I understand that you try to make things easier, but you might actually make things worse. As meta tags aren't visible, site owners might have trouble to understand, what's going on.
