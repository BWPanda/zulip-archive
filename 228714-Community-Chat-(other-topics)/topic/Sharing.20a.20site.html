klonos (Sep 23 2020 at 13:19):

@Kevin Golding what you need for any Backdrop site is:

  • its codebase
  • its database
    Once you have provided people these two things, it's as simple as copying the codebase + importing the database.
    Beyond that, you can make this as complicated/involved as required. For example, if no custom or contrib modules have been used, you basically only need the /files directory (which includes the configuration and media assets) instead of the entire codebase. You can then instruct users to download and install the same version of Backdrop from scratch (instead of copying the entire codebase).

klonos (Sep 23 2020 at 13:21):

https://api.backdropcms.org/documentation/deploying-backdrop-site could possibly be simplified, as the files directory is usually included in the codebase, and then also the configuration directory is included in the files directory. So unless you've deviated from the defaults, you only need the codebase and the database.

klonos (Sep 23 2020 at 13:37):

I've updated the documentation in https://api.backdropcms.org/documentation/deploying-a-backdrop-site to include a simple/default scenario, and an advanced scenario. I hope it makes sense.

indigoxela (Sep 24 2020 at 07:01):

it appears the configuration folder name is derived from a hash of the database connection string, and thus will change when moving to a new server.

Not necessarily. The config directory names are defined in settings.php. Whatever you set there will work. You can change that, but it won't get overridden if it's already set. Make sure that the config directory ships with your export.
