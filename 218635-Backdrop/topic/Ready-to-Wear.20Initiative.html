cellear (Luke McCormick) (Jun 12 2020 at 01:58):

Ah, I would call that "steps for the initiative process." I answered the opposite branch.

Your list looks pretty good; I was envisioning something like this:

  1. Figure out what attributes differ between themes.
  2. Find examples of existing themes across all platforms that exhibit different attributes
  3. Look for low-hanging fruit. Existing Backdrop themes? D7 themes? Wordpress Themes? HTML or other environments?
  4. Of the non-low-hanging fruit, which conversions seem to be plausible?
  5. What mechanisms do we want to provide to demo themes? Sample content? Suggested layouts? Tugboat?
  6. How do we test the themes?
  7. Maintenance. Once the MPV is released, how do we keep it from getting stale? How do we maintain the themes, add to them, retire them?
  8. How do we want to package and "market" the results?

That's by no means a final or definitive list, it's just meant as a starting point.

I'm just listening to the dev meeting (first time I've listed to a meeting in a while) and towards the end you're talking about why Tugboat takes longer now than before to spin up a demo site. I think one of the reasons is that previously it would start a job to build a preview site, then wait 3 seconds and check if the job was complete, otherwise repeat. Now however, Tugboat provides a 'Retry-After' header which specifies how long you should wait before trying again. It's currently (or was when I was doing all this a few months ago) set to 10 seconds. I asked the Tugboat guys about this and while they admitted it was a long time to wait (for something that should be fairly quick), they also said that ignoring this 'retry-after' time would technically be hacking their API and advised against it. So that's why it takes longer. My impression is that it's already possible to create a good-looking site without needing custom code; it's just not obvious how to do it. So that reduces R2W to two distinct steps:

2) See which parts of that process can be optimized and streamlined

cellear (Luke McCormick) (Oct 09 2020 at 16:34):

Step 1 includes:

  • Figuring out what you're trying to do
  • Selecting a theme (including finding, previewing, installing, and configuring)
  • Choosing/configuring/creating layouts
  • Color choices (and possibly other global visual configurations)
  • Modules/systems for things like carousels
  • Content creation (for example heroes/icons/backgrounds)

  • Color options, storing and retrieving
  • Config scripts
