StPaulTim (Jan 26 2021 at 03:23):

My understanding is that we have different repos, because of permissions. It was my understanding that permissions in the issue queue are tied to permissions for the repo and we would either have to be very conservative in granting permission for managing issues or be very liberal in granting access to the code.

I think that was the original reason for two different repos. I don't know if Github has made it easier for us to accommodate both.

jenlampton (Jan 26 2021 at 04:22):

It is easier to manage now than it was before, but it's still not ideal. The main reason for not combining now is that we'd have a problem with issue numbers (since PRs are issues too) we'd have conflicts merging the two repos.

jenlampton (Jan 26 2021 at 04:23):

I think if we're going to declare PR bankruptcy and close/delete them all, we should do that at a nice clean milestone, like 2.x.
