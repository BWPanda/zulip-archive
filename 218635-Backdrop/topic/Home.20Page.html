even for experienced Drupal users where every single one of them is expecting to have to go change it. Instead, it should very rarely be changed because the OOTB experience is the most ideal for most situations

@Phil Ward - I know what your talking about in Drupal 7, but I think that the Backdrop CMS layout system largely addresses this issue for me. I seldom change change the home page path anymore, because the home page layout is so flexible that I can do almost anything I want with it.

If I want a node on the front page, I drop a node block onto the front page. If I want a view on the front page, I drop a view block onto the front page layout. I have a hard time thinking of a reason why I would want to change the default home page in Backdrop (unlike Drupal 7).

StPaulTim (Dec 29 2020 at 20:48):

Now, I would agree - that this might not be obvious to users coming from Drupal 7. Wrapping my head around layouts was the most difficult thing for me while adjusting to Backdrop. But, having used the layout system for a while, I really like having a home page layout and don't see why I would change the default home page (which I think would be confusing to users later on).

StPaulTim (Dec 29 2020 at 20:49):

Having said that, I think it is unnecessarily confusing for users that choose to change their default home page.

Phil Ward (Dec 29 2020 at 21:16):

@StPaulTim right. I feel like it's half-way there but the experience of "what to expect" is what is kind of throwing things off.

I think if we can address the problem where single page layouts are living in the same space as multi-page layouts, this will become a lot more clear for most people.
