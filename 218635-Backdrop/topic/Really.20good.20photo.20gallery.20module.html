mazze.ch (Dec 11 2020 at 09:19):

Hi @Elisabeth Garbeil this here might not be a direct solution to your problem... but I was fiddling around with image galleries for years in Drupal 7. Now in Backdrop, I use the Uikit CSS framework (not Bootstrap, like core does). But this forced me to write field templates, which is what I should have learned years ago. This is my solution for an image gallery... 9 lines of code... and the backend is a pure Backdrop CMS image multifield, including sorting, crop, caption

<div class="uk-child-width-1-4@m uk-child-width-1-3@l uk-child-width-1-4@s uk-child-width-1-4 uk-grid-small" uk-grid uk-lightbox="animation: slide">
<?php foreach ($items as $delta => $item): ?>
<div>
<a class="uk-inline" href="<?php print image_style_url('16_9_standard', $item['#item']['uri']); ?>" data-caption="<?php print render($item['#item']['title']); ?>"><img src="<?php print image_style_url('16_9_preview', $item['#item']['uri']); ?>" /></a>
</div>
<?php endforeach; ?>
</div>

I do not know Bootstrap enough, but I assume something very similar should be possible there, or with another 3rd party script. It's not a module solution, but I (being a marketer as my main profession) would never, never go back to any module-based solution for image galleries.

Phil Ward (Dec 13 2020 at 20:32):

There are three points of consideration when dealing with images:

1) Backend where the images are added

2) Frontend where the images are viewed

3) Frontend where the images are interacted with

One of my biggest complaints about Drupals handling of images is the lack of congruity between the Backend and Frontend interaction.

Every Backend module or frontend viewing module must "include" support for the frontend interaction and many times this does not happen without a feature request.

I have wished for a long time that "Drupal" would create a better API around the interaction modules where a module can register itself as an interactive module to be used by any viewing module and it becomes available to the system from the time it is enabled. I think this is something Backdrop can look into fixing for the long term. (Maybe it's already happened as I don't have enough hands on experience yet)

When newer interaction modules are added, they are not always supported by older viewing modules. This has caused me to drop one module and search for a different one because they don't all work from start to finish. Point is, when it comes it image manipulation, a better job could be done to seamlessly allow them to know about and talk to one another.
