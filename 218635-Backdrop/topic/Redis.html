Please read through https://backdropcms.org/philosophy if you haven't done so already. Special note to the 1st and top-most of our principles noted there:

  1. Easier updates: Backwards compatibility is important. Backdrop will attempt to keep API change to a minimum in order for contributed code to be maintained easily, and for existing sites to be updated affordably. Every big change that can't be made backwards compatible will need to be carefully considered, and measured against Backdrop's other principles:
