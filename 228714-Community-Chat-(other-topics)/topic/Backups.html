Elisabeth Garbeil (Oct 26 2020 at 22:07):

BWPanda said:

I believe that means I need encryption... Is that correct, or is there another way to prevent someone with access to your files from reading them?

yep, you need encryption and you might run into issues with bandwidth if you're doing remote backups to a cloud server, so look at that very closely if you do decide to go that route. Another way to do it if you're planning on leasing a server is to have the files there all the time (think github) and just connect when you need them and save them back to the server. You can purchase a backup product when you lease the server - many come with.

Elisabeth Garbeil (Oct 26 2020 at 22:09):

BWPanda said:

One of my other requirements (well, not so much as a requirement as a nice-to-have) is that I'd like to be able to browse my files on the backup server and copy them back to my laptop (or another device) without having to perform a full restore.

This really depends on how your backup is set up. If the files are backed up to a single file, compressed and then encrypted - then you will have to download the whole backup and unencrypt it before finding your file.

Elisabeth Garbeil (Oct 26 2020 at 22:12):

I think it also depends on what files you're trying to backup. You may want a combination of solutions - for example an external drive that you use to backup system files so that you can restore your linux box if anything (say a hard drive) dies and a separate backup for the projects you're working on.

For example, I use an external hard drive to backup my laptop nightly (automated), but we have separate leased servers where our clients sites and development reside and those are in a six sigma data center with an automated backup solution.

Elisabeth Garbeil (Oct 26 2020 at 22:16):

Well backing up to a single file is obviously not what I want to do then :smile: Is there another way?

There's any number of combinations, depending on the solution you go with. Many backup softwares have the ability to search within a back file and restore a single file from an encrypted backup. I don't have a linux system, but you could cruise the net and see what the best backup options are. If you are planning on leasing a server - Don't go with GoDaddy. They're the largest, but they don't provide good service and will nickel and dime you to death (it gets very expensive very quickly). Depending on where you're located, there are good hosting providers. We use Liquidweb and used to use MediaTemple before they were sold. Cloud servers are a good option for smaller requirements and less maintenance.
