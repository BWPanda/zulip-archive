Topic: Can't clone the default layout?

{% raw %}

Jeff Porten (May 12 2020 at 01:54):

Just made a mistake I've made a half-dozen times: wanted to modify the layout for a particular page with different blocks, and accidentally did all the setup on the default layout. What would be nifty here: being able to clone the default layout into a new one, set the page for the new one, and then fix the default. It's easier to revert a layout back to the original than it is to create a new layout and put everything in its place. But the default layout(s) appear to be the only ones that _omit_ a clone button?

klonos (May 12 2020 at 03:31):

There's a request for that: https://github.com/backdrop/backdrop-issues/issues/1674 :wink:

[UX] Add the ability for default layouts to be cloned (like views). 路 Issue #1674 路 backdrop/backdrop-issues
Simple: a user might not want to start completely from scratch and it might be more convenient for then to clone an existing layout and edit it.

klonos (May 12 2020 at 03:32):

Yeah, I am always playing around with the default ones, so I forgot about this. What I mean is that if one customizes the default ones and needs to clone them at that stage, then they should have that option. Is there any reason why we do not allow the default layouts to be cloned?

klonos (May 12 2020 at 03:33):

^^ exactly the situation I gotten myself into a few years back when I was fooling around with layouts :slight_smile:

Jeff layouts are all config, so just copy the default.json file, change these bits:

  "_config_name": "layout.layout.default",
  "path": "",
  "name": "default",
  "title": "Default Layout",

and then reset the default.
