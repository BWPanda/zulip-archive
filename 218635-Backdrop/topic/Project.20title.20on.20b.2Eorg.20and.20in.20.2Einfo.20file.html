On Backdrop Modules, is the listed name of a module taken from the .info file or from the title of the README.md file? And should the title in those two places always be the same?

I ask because I observe that on drupal.org/projects, the project page title isn't necessarily the same as the module title in its .info file. For example, Project title = "Ubercart Price Per Role", but module title in .info file is "Price per role".

In that case, it seems to make sense to have "Ubercart" in the title on the project page, but not in the module listing on the website (capitalization ought to be the same, though). Wondering if a similar situation is possible (and/or recommended) with BD's project listing.

klonos (Dec 30 2020 at 22:28):

...but module title in .info file is "Price per role".

It shouldn't be that way. Coding standards mandate that project names be capitalized as a proper name (see the "Properties" section under "More about module .info files" in https://api.backdropcms.org/modules):

This displays the name of your module, which will appear on the Modules page. Since module names are proper names, it should be capitalized as a proper name (e.g., "Dragon Builder", not "dragon builder" or "Dragon builder"), and it should be a human-readable name (not dragon_builder). If your module name includes an acronym (CSS, WYSIWYG, UI, etc.) or a third-party trade name (jQuery, JavaScript), of course follow the standard capitalization for those.

So the name in the .info file should be "Price Per Role"

klonos (Dec 30 2020 at 22:50):

...consider writing the following in a documentation page for example:

...you need to install some additional modules like Markdown Editor for BUEditor.

In the above sentence, one might read "Markdown Editor" and "BUEditor" as being 2 separate modules. Whereas if the module name was written in proper case, it would instead be:

...you need to install some additional modules like Markdown Editor For BUEditor.

...which makes it clear that this whole 4-word thing is a single module.
