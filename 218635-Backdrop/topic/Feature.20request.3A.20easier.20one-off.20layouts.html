Jeff Porten (Oct 26 2020 at 06:16):

I just set up a custom layout for a particular page, and once again had to consult documentation and jump around between half a dozen pages before I figured out what I had gotten wrong. (This after I accidentally put new blocks on my entire site because I didn't realize I was editing the default page.)

There are a couple places where things can happen automagically in Backdrop, and this strikes me that it should be one of them. How about a contextual menu on a page, or a new tab on the Layout editor, that says "Enable custom layout for this page/the referring page". That should clone the existing layout, set it to only work on the existing path, maybe name it [Referring Page Title] Layout, and drop me back into the Layout editor.

Even better: when I go to the Layout editor from a particular screen, why not put a drop menu next to the Layout title so I can switch layouts on the fly? Put "add new..." on the bottom and life is easy.

I'll add, FWIW, that every time I have to do this, I completely forget that I want the Path field to be node/% and that the specific path is a Visibility condition.
