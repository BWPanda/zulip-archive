bugfolder (Feb 23 2021 at 17:12):

Has the topic of supporting wildcards in layout paths ever come up? Not % placeholders; true wildcards, so that, for example, a layout with path foo* would be used for both /foospam and /fooeggs. Particularly if implemented along with issue 3749, Make layouts aware of path aliases, this seems like it would be really useful. One could create a layout with path *, then use visibility conditions in "Menu settings" to apply it to all sorts of diverse paths selectively.
