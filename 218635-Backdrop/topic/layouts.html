StPaulTim (Dec 29 2020 at 06:50):

For the record, our current use of the words is something like this:

  • Layout Template = A specific configuration of regions. Boxton and Geary are layout templates. Layout templates are not specific to paths, they are only a specific configuration of regions.
  • Layouts = All of the specific blocks and configuration (visibility rules, etc) attached to a specific template. Your home page layout might use the Boxton layout template but still be different from other layouts that use Boxton, based upon the blocks, content and configuration.

StPaulTim (Dec 29 2020 at 06:53):

We've already made a number of changes to the UI to reflect these specific definitions and with the intent to reduce confusion.

klonos (Dec 29 2020 at 23:29):

Well, if you think of it this way it may make more sense:

  • layouts are NOT pages
  • layouts are set of regions + rules that dictate how one or more pages will be rendered (what blocks/content will be shown/hidden and where, and in which conditions).
  • the fact that there may be a case where a layout applies only to a single page on a site should never be considered that the layout is the page (or the other way around for that matter)

@klonos I'm going to have to disagree with:

layouts are not pages

If there is a path to a specific location that is output for general view, it is a page. It's less about how it is output and more about how it is perceived by the end-user viewing it or building it.
