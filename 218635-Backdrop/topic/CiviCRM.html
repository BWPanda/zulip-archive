Backdrop shares much of the deep integration with CiviCRM that Drupal 7 offers. ericfg (Mar 29 2020 at 20:28):

late to the discussion:
yes Civi is a great tool and for anything related to managing events, people, contracts, interatcions, etc. it is the tool to use (of course with backdrop as the front-end, or well, drupal 7)

civi is UGLY, for sure. Shoreditch does a ton to start moving that in a better direction.

For me, when I build complex stuff with civi (d7/backdrop), as much as possible I build views with the relevant data for various funcitons/actions/work tasks. If possible I shield the end-user from the ugly of civi, while getting all the great out of it as a CRM.
sure there are some levels of staff/actions/etc. that can't be done via views. those folks need to use the ugly civi interface, but they get used to it pretty fast and the first time having all their data in civi makes some crazy task really simple to pull off, they start to love it even if it's clunky and old looking.

ericfg (Mar 29 2020 at 20:35):

an example: an organization does recruitment events. a default setup would have them enter the event as an activity attached to the school it happened at, fill in some fields custom to that activity (how many students showed up) and then add a relationship from the activity to the people from the organization that did the recruitment event.
It's a five or so step task through different interface screens; would require a ton of training. Most of the data is entered by relatively low-skilled interns, they are good at paying attention to the data but really not at following complex workflow steps to find where to enter data.
so, instead it's a civicrm enabled webform.
All the information necessary is on one page/form.
Then they submit and a function handles all the various tasks and they end up on a page that is views that are summaries of this year's recruitment events.
Data entry folks never need to touch civi, but all the data is stored there and usable for tons of different things; and the higher level admin staff have become comfortable enough with civi to use it to look at and act on the data coming in.

Attn CiviCRM - Backdrop folks. Just released a rough/early version of a module that I think has some good potential. Looking for a simple way to allow some deeper integration between CiviCRM and Backdrop functionality. It's based off the Group-Role sync module and has been modified to sync contacts in CiviCRM groups with Backdrop nodes (of a specific content type set in the module configuration).
https://github.com/backdrop-contrib/civicrm_group_nodes

backdrop-contrib/civicrm_group_nodes
Sync contacts in a CiviCRM group to Backdrop nodes. - backdrop-contrib/civicrm_group_nodes

Laryn (Feb 17 2021 at 17:03):

I'd love some outside testing and feedback.

Laryn (Feb 17 2021 at 17:04):

From the README:

This will allow a variety of references between Backdrop functionality and CiviCRM contacts via this relationship. For example:
- Automatic landing pages with friendly URLs for CiviCRM Contacts (through the referenced node).
- Enabling Backdrop comments on a contact node.
- Allowing Backdrop fields (such as Term References or any other) to be associated with a CiviCRM contact.
- Using third-party modules like FiveStar to allow ratings in Backdrop which are associated with a CiviCRM Contact.
