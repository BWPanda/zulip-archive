Phil Ward (Jan 27 2021 at 05:59):

Would it be worth discussing the idea to add a way to pull PRs directly from Git within the GUI of core? Basically a Search for the PR then install it in the current environment, all without leaving the browser.

There are a lot of PRs that need tested, and there are a lot of PRs that are 85 - 99% there, but just haven't been committed and it would be nice to either use the PR as-is on a live site until it gets added to a release, and(or) be able to do testing on specific PRs that need testing and feedback.

Yes, I know PRs can be pulled through CLI or manually downloaded, but a lot of folks won't mess with it because they don't know how or it's too involved. But if they could visit a page in core and type a git PR number and have it installed for them within say 15 seconds of finding the PR number, I think we would see a lot more people engaging in testing of the PRs.

I could see two modules to handle this: 1) PR Installer module and 2) Factory Reset module. The first one would handle the communication with Git, searching for PRs, showing which PRs are installed etc. The second one would be used to completely wipe the site's database and put it back to a fresh state like right after the install profile was finished.

It would be a bit of upfront work to get it going but I think the long-term payoff could be worth it.
