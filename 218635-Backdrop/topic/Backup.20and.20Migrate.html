cellear (Luke McCormick) (Nov 27 2020 at 16:34):

My interest in BaM stems from the challenges I've faced creating a local copy of a site I'm working on in the Simplo hosting environment that @StPaulTim is developing. (Which, by the way, led me down this rabbit hole:

https://dev.mysql.com/doc/refman/5.7/en/sql-mode.html

"It is preferable to create MySQL accounts with CREATE USER rather than GRANT. NO_AUTO_CREATE_USER is deprecated and the default SQL mode includes NO_AUTO_CREATE_USER. Assignments to sql_mode that change the NO_AUTO_CREATE_USER mode state produce a warning, except assignments that set sql_mode to DEFAULT. Expect NO_AUTO_CREATE_USER to be be removed in a future release of MySQL, and its effect to be enabled at all times (and for GRANT not to create accounts any longer)."

Which was addressed here:

https://github.com/backdrop/backdrop-issues/issues/4238

But: I discovered it when I I found I wasn't able to create new Backdrop sites in Backdrop 1.17.1, which was released Sept 30. Apparently my new Ubuntu system is pretty aggressive about upgrades, and had updated MySQL to 5.8.

This probably happens to you guys all the time, but I found it pretty shocking to have what seemed like correct new-site technique met with this:
