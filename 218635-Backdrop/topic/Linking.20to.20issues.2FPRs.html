BWPanda (Jan 28 2020 at 19:59):

So a while ago I setup Linkifiers in Zulip for automatic linking to GitHub issues and PRs. I had an issue with linking to any repo with a . in the name (e.g. github.com/backdrop-ops/backdropcms.org), so I sent an email to Zulip help. They just replied (apologising for the delay) and explained that . does indeed work in the Linkifier regex, I was just doing it wrong: I had [a-zA-Z0-9_-.] which it took _-. to mean any character between _ and . which isn't valid, so I couldn't do it. Zulip support pointed out I should simply do [a-zA-Z0-9_.-] instead which of course works. So I've updated the Linkifier regex and now you can link to any repo's issue/PR!
