StPaulTim (Apr 30 2020 at 20:21):

a tool like Zulip which intends for people to read it later (unlike online chat, which is intended to be real-time)

See, I don't see that difference in Zulip and Slack. I believe that they are both intended to be used in real-time like chat. The difference is that some people want to go back and see what they missed in the real-time discussion and to read several days of the real time discussion (regardless of whether that is the intention or not). klonos (Oct 25 2020 at 18:25):

@Tuhin we have only recently changed our chat platform, so we cannot simply switch the entire community over again. The main reason we witched from Gitter to Zulip was that the community was getting bigger, and we needed a chat system that supported channels/threads, in order to better handle the increased activity. ...IF we are to change again, there will need to be good reasons. ...here's what I would do if I were you:

  • please open an issue in https://github.com/backdrop-ops/backdropcms.org/issues
  • list the problems with Zulip
  • list the things that discord does better

  • ask for feedback and support for the switch from other Backdrop community members
    ...while at it, please go through the list of features we need, and make sure that Discord does all of these:

  • discussion threads/topics

  • automatic issue linking from GitHub
  • API to support sending notifications from GitHub and our forum to it
  • chatbot support
